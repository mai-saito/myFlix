myFlix

This app will provide you information about different movies, directors, and genres. You will be able to sign up, update your personal information, and create a list of your favorite movies.

Description Endpoint URL Type Request body Response body
Get a list of all movies /movies GET None JSON object holding data about all the movies, containing the title, genre and director's name property. Example:
{
"Title": "The Greatest Showman",
"Description"....":
"Genre": "Musical",
Name :"Michael Gracey"
},
Get data about a single movie by title /movies/[Title] GET None JSON object holding data about a single movie.
Get data about a movie genre by movie title /movies/genres/[Title] GET None JSON object holding data about a movie genre.
Get data about a director by name /movies/directors/[Name] GET None JSON object holding data about a director.
Add a new user /users POST JSON object holding data about the user to add. Example:
{
"Username": "Paulina Gomez",
"Password": "12345",
"Email": "paulina@gmail.com",
"Birthday": "2000-01-01"
} 		JSON object holding data about the user that was add, including an ID. Example:
{
"Favorites": [],
"_id": "5d7cc5e9a3d1dc4290f30036",
"Username": "Paulina Gomez",
"Password": "12345",
"Email": "pauilna@gmail.com",
"Birthday": "2000-01-01T00:00:00.000Z",
}
Update a user information /users/[Username] PUT JSON object holding data about the user to add. Example:
{
"Username": "Paulina Gomez"(required),
"Password": "123456789"(required),
"Email": "paulinagomez.gmail.com"(required),
"Birthday": "2000-01-01"
} 		JSON object holding data that was newly updated.
Add a movie to the list of favorites /users/[Username]/Movies/[MovieID] POST JSON object holding data about a movie which users like to add to their list of favorites. JSON object holding data about the movie that was added to the list of favorite.
Delete a movie from the list of favorites /users/[Username]/Movies/[MovieID] DELETE None A text message indicating the movie that was removed from the list of favorites.
Remove a user /users/[Username] DELETE None A text message indicating the user who was removed
